  - `electron/main.js` に `focus-webcontents` IPCハンドラーを追加（`mainWindow.hide() → mainWindow.show()` でWindowsに強制再アクティベーション）
  - `electron/preload.js` に `focusWebContents` API を追加

### 2. 「次の週を作成」の日付ロジック修正
- **症状**: 5/11〜5/24の工程表で「次の週を作成」を押すと 5/25〜6/7（2週間丸ごと先）になっていた

## 未解決の問題

### セル縦幅自動拡張（同一行の追従）
- **症状**: セル内でEnterで改行すると、そのセルは伸びるが**同じ行の他のセルの枠が伸びない**（見た目がズレる）
- **試した修正**: `RichTextCell.jsx` でラッパーを `height: rowHeight` → `minHeight: rowHeight` + `height: '100%'` に変更
  - `minHeight` だけだとセル自体は伸びるが横並びのセルが追従しない
  - `height: '100%'` を追加したが、CSSの制約上 `<td>` の計算済み高さが子要素に伝播しないため未解決
- **根本原因**: HTML tableの `<td>` は計算済み高さを `height: 100%` の子要素に伝播しない（CSS仕様）
- **有力な解決策（未実装）**:

#### 案A: ResizeObserver で rowHeights を動的に更新（推奨）
```jsx
// RichTextCell.jsx に追加
// ラッパーの高さを監視し、rowHeight を超えたら親に通知
const wrapperRef = useRef(null)
useEffect(() => {
  if (!wrapperRef.current || !onRowHeightChange) return
  const observer = new ResizeObserver(entries => {
    const h = entries[0].contentRect.height
    if (h > rowHeight) onRowHeightChange(h)
  })
  observer.observe(wrapperRef.current)
  return () => observer.disconnect()
}, [rowHeight, onRowHeightChange])
```
- `ScheduleGrid.jsx` → `RichTextCell` に `onRowHeightChange={(h) => onRowHeightChange(ri, h)}` を渡す
- `App.jsx` の `handleRowHeightChange` がすでに存在するので、これ経由で `rowHeights` state を更新
- rowHeights が更新されると全セルが同じ `minHeight` で再描画される → 行全体が揃う
- Firestoreにも自動保存される

#### 案B: ScheduleGrid で実際の `<tr>` 高さを計測して同期
- `<tr>` の `ref` を使って実際の高さを計測し、`rowHeights` を動的更新する
